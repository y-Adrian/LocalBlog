## 1.1 一、每天 3 小时怎么用（建议固定结构）

| 时段 | 时长 | 内容 |
|------|------|------|
| A | ~40 min | 复习：昨天错题 / 模板默写（二分、拓扑、并查集、背包） |
| B | ~95 min | 新题：优先独立完成；卡 **15 min** 再看 hint / 题解一小段 |
| C | ~45 min | 总结：暴力 → 优化点 → 复杂度 → **同类题标签**（四句话） |

**题量**：多数天 **2 题 Medium**；标「×3」为 **3 题**或 **2 题 + 1 二刷**。

**原则**：某天严重超时 → **砍掉第 2 题**，把剩余时间用于 **第 1 题二刷默写**，对薄弱项更有效。

## 1.3 三、每题错题本最小字段（复制成模板即可）

在笔记里每题一行或一页，建议字段：

1. **题号 + 标题**  
2. **标签**（图 / 并查集 / DP-背包 …）  
3. **第一次**：是否独立 AC？耗时？  
4. **思路四句话**：暴力 → 优化 → 复杂度 → 相似题  
5. **二刷日期** + 是否默写通过  

二刷间隔建议：**3～7 天**。

## 1.4 四、推荐：笔记与打卡怎么做（好用的几种）

### 1.4.1 Notion / 飞书文档（最省心）

- 用 **Database**：列 = `日期 | 周主题 | 题号 | 独立AC | 二刷 | 复盘链接`。  
- 每天一条记录，勾选 + 贴题链接；周视图一眼看出空档。  
- 适合：想轻量、少折腾、手机也能勾一下。

### 1.4.2 Obsidian（适合程序员、本地优先）

- 每天一篇 `2026-05-13.md`，内嵌本计划的「当天那一行」+ 做题日志。  
- 用 **Dataview 插件** 可把「未完成二刷」自动聚合成列表。  
- 适合：喜欢 Markdown、希望笔记和计划都在本地 Git 里。

### 1.4.3 Excel / Google Sheets（最直观）

- 一行一天，列：`完成 | 题1 | 题2 | 耗时 | 备注`。  
- 加条件格式：未完成标红；**连续打卡天数**用简单公式也能做。  
- 适合：只想看表格进度条、不做复杂笔记。

### 1.4.4 LeetCode 自带 + 外部极简

- LeetCode **题单 / 收藏夹** 按周建文件夹；做完打星标「要二刷」。  
- 复盘长文写在 Notion/Obsidian 任意一处，题面只留链接。  
- 适合：讨厌多工具，一切以题为锚。

## 1.5 五、Hard 与投递节奏（提醒）

- **Hard**：不必堆量；每周 **0～2 题**，优先 **并查集 / 图 / 堆 / 二分答案** 类经典即可。  
- **约第 4～5 周起**可少量投递练手，用真实面试调整「图 / DP / 并查集」周内的比例。
