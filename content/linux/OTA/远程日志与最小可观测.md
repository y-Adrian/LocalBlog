# 远程日志与最小可观测

本文对应 [[成长路径/index|成长路径]] **OTA / 运维**：在无重型 K8s 的嵌入式设备上实现 **可运维** 最小集。

---

## 日志

| 方案 | 说明 |
|------|------|
| **rsyslog → 中心** | 成熟，需网络 |
| **MQTT / HTTP 批量上传** | 适合弱网，本地缓冲 |
| **旋转文件 + scp** | 现场维护 |

控制 **体积**：`logrotate`、只保留 ERROR 上云。

见 [[linux/文件系统/日志持久化与 barrier]]。

---

## 指标（最小）

- **心跳**：进程定期写文件或 HTTP ping。
- **资源**：`/proc/meminfo`、CPU、`df` 周期采样。
- **业务**：包计数、错误码（与 DPDK `xstats` 可对照）。

无需一上来 **Prometheus**；可先 **JSON 行协议** 上报。

---

## 健康检查

- **watchdog** 硬件 + 用户态喂狗。
- **systemd Restart=** 或自定义 supervisor。

---

## 安全

- TLS 上传、设备 **证书** 轮换策略。
- 日志脱敏（无密钥、无 PII）。
