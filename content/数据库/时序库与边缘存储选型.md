# 时序库与边缘存储选型

本文对应 [[成长路径/index|成长路径]] **低优先级**：设备侧 **指标/日志** 持久化时，何时用 **SQLite**、何时用时序库或 **云端**。

---

## 数据类型

| 类型 | 例子 | 常见存储 |
|------|------|----------|
| 配置 | 键值、拓扑 | [[数据库/SQLite 嵌入式配置库]] |
| 指标 | CPU、PPS、温度 | 时序库 / 本地 TSDB |
| 日志 | 文本行 | 文件、Loki、云端 |
| 事件 | 告警一条 | SQLite / 消息队列 |

---

## 边缘常见方案

| 方案 | 适用 |
|------|------|
| **仅文件 + 轮转** | 最简单，查询弱 |
| **SQLite + 时间索引表** | 小规模历史 |
| **TDengine / Influx 边缘版** | 指标多、聚合查询 |
| **上云 only** | 网络稳定、存储在中心 |

---

## 选型问题清单

- [ ] 单机保留 **多少天** 数据？上限 **MB/GB**？
- [ ] 是否需要 **降采样**（分钟/小时聚合）？
- [ ] 断网时 **缓冲** 多久？是否压缩上传？
- [ ] 与 [[linux/OTA/远程日志与最小可观测]] 架构是否一致？
